# 当前界面设计规范

## 1. 文档作用

这份文档用于记录当前作品集站点已经落地的界面规范，覆盖：

- 首页 `index.html`
- 案例页通用框架
- `digimi.html`
- `digistudio.html`

目标不是写抽象设计原则，而是把已经验证过、应继续沿用的真实规则固定下来，避免后续迭代把页面重新做回“重卡片、重装饰、留白失衡”的方向。

---

## 2. 整体视觉基调

### 2.1 关键词

- 克制
- 清晰
- 偏编辑器感
- 白底、高级冷灰、深蓝强调
- 有动效，但不炫技

### 2.2 全站原则

- 页面背景统一使用白色，不偏黄、不偏米。
- 版心偏窄，左右留白要充足，但不能让正文显得发飘。
- 文字层级清楚，避免整页都是超大标题。
- 图片是项目页里的主信息，文字负责解释和组织节奏。
- 卡片只在真的需要承托图片或图片+说明时出现。
- 动效只做轻量增强，不做强视差、旋转、复杂连续动画。

## 3. 全局设计 Token

当前主色和语义色已经固定在 [styles.css](/Users/yezehong/Documents/Codex/2026-05-31/files-mentioned-by-the-user-project/portfolio-site/css/styles.css) 顶部：

```css
:root {

  --color-body: #475569;
  --color-muted: #94a3b8;
  --blue-deep: #1e40af;
  --blue-light: #60a5fa;
  --gradient-primary: linear-gradient(135deg, #1e40af 0%, #60a5fa 100%);
  --bg-pure: #ffffff;
  --bg-subtle: #f8fafc;
  --bg-card: #ffffff;
  --border-hairline: 1px solid #e2e8f0;
}
```

### 3.1 颜色使用规则

- `--color-title`：主标题、重点小标题、强调段落
- `--color-body`：正文、导语、说明
- `--color-muted`：弱提示、辅助说明、引用/原声
- `--blue-deep`：主强调色、激活状态、编号、重点标签
- `--blue-light`：辅助蓝、hover 强化、渐变终点
- `--bg-subtle`：浅灰底板、内容承托面
- `--border-hairline`：极细分隔线，不使用厚重描边

### 3.2 背景纹理规则

`digistudio` 方向已经加入轻量画布感背景：

```css
.canvas-grid-bg {
  background-image: radial-gradient(#e2e8f0 1.5px, transparent 1.5px);
  background-size: 16px 16px;
}
```

使用原则：

- 只作为大区块或编辑器相关容器的弱提示
- 密度要低，不能抢正文和图片
- 目的是暗示“Canvas / 编辑器”，不是做装饰花纹

## 4. 排版与版式规范

### 4.1 版心

- 首页和案例页都采用偏窄版心。
- 首屏文字区、章节导语区、正文区要尽量共享同一套宽度逻辑。
- “留白大”不等于“文字区域窄”。正文和导语必须有足够宽度，避免频繁断句和突兀换行。

### 4.2 标题层级

- 大标题用于定义项目主题，数量要少。
- 小标题负责组织章节，不要每一行都写成“汇报式大标题”。
- 页面里标题和导语优先做通栏，避免默认拆成左右两栏后出现大片无效留白。

### 4.3 字号和节奏

- 标题允许有明显层级，但不能压住整页呼吸感。
- 正文以易读为先，长度适中，避免段落过满。
- 当前项目页正文已经压缩过一轮，后续继续写文案时，应优先控制在“够清楚，但不过度解释”。

当前项目页统一采用以下字号表，后续新页面优先按这套数值执行：

| 层级 | 桌面端 | 平板端 | 手机端 | 建议用途 |
| --- | --- | --- | --- | --- |
| 项目主标题 `H1` | `56px / 3.5rem` | `48px / 3rem` | `36px / 2.25rem` | 首屏项目标题、页面主命题 |
| 章节标题 `H2` | `40px / 2.5rem` | `34px / 2.125rem` | `30px / 1.875rem` | 章节标题、通栏主句 |
| 小标题 `H3` | `20px / 1.25rem` | `20px / 1.25rem` | `18px / 1.125rem` | 分段标题、结构化小节标题 |
| 首屏副标题 / 导语 | `20px / 1.25rem` | `18px / 1.125rem` | `17px / 1.0625rem` | 首屏副标题、章节导语、总结引导句 |
| 正文 | `16px / 1rem` | `16px / 1rem` | `15px / 0.9375rem` | 正文段落、说明、要点描述 |

- 标题默认字重保持在 `700` 左右，不靠过粗来制造层级。
- 项目页里真正的大标题只保留在首屏，正文区域优先使用 `H2` 和 `H3`。
- 如果某一段标题看起来过重，优先缩短文案，再考虑放大字号。

## 5. 卡片与容器规范

### 5.1 卡片使用边界

项目展示页统一遵循以下规则：

- 首屏不需要卡片。
- 仅展示文字内容的时候不需要卡片。
- 正文中的图片，或者图片和文字一起展示的时候，才需要卡片。

### 5.2 卡片风格优先级

- 优先使用高级感的浅灰色底板。
- 默认优先无边框。
- 如果需要边界感，先用极细浅灰线，不先上厚描边或重阴影。
- 不再使用“卡片里面套卡片”的堆叠式做法。

### 5.3 图片组补充规则

- 如果是一组大小图片垂直排列，上方图片可以加很浅的描边。
- 同组中下方图片不需要描边。
- 这类差异只用于帮助层级，不作为装饰语言。

### 5.4 编辑器感皮肤化细节

`digistudio` 已明确采用“轻编辑器皮肤”：

- 1px 细线边界，参考代码编辑器或游戏编辑器外壳感
- 极淡网格点背景
- 冷灰底板
- 低饱和、大扩散阴影

目标是让外围网页和页面中的编辑器截图属于同一个视觉语境。

## 6. 动效规范

### 6.1 全局原则

- 动效用于增强浏览流动感，不用于制造炫技感。
- 所有动效都应低幅度、低噪音、低干扰。

### 6.2 当前已确定的动效类型

- 首屏和导航：轻微淡入上移
- 模块进入：滚动 reveal
- 项目卡片 hover：轻微上浮、阴影增强、图片轻度放大
- 首页项目卡片：鼠标 hover 时显示跟随鼠标的“查看更多”悬浮提示
- `digistudio` 方案区：Tab 切换时用轻淡入切换，不做花哨转场

### 6.3 明确避免

- 强视差
- 连续旋转
- 复杂粒子
- 长时间自动运动
- 高对比闪烁

## 7. 首页规范

参考 [index.html](/Users/yezehong/Documents/Codex/2026-05-31/files-mentioned-by-the-user-project/portfolio-site/index.html)、[script.js](/Users/yezehong/Documents/Codex/2026-05-31/files-mentioned-by-the-user-project/portfolio-site/js/script.js) 与 [styles.css](/Users/yezehong/Documents/Codex/2026-05-31/files-mentioned-by-the-user-project/portfolio-site/css/styles.css)。

### 7.1 首屏

- 首屏文案区宽度要和下方内容区统一。
- Hero 采用居中结构，不再保留旧的附加侧栏。
- CTA 保持轻量，不做复杂按钮系统。

### 7.2 代表项目区

- 标题区与瀑布流区域共用同一套宽度逻辑。
- 标题和导语不能只缩在左栏，需和下方内容宽度统一。
- 两列瀑布流保持偏紧凑，不要拉得过宽。
- 两列之间间距偏克制，避免出现中间大沟。
- 卡片整体尺寸收敛，两侧留白比早期版本更大。

### 7.3 项目卡片

- 图片在上，文字在下。
- 图片宽度和文字区宽度必须对齐。
- 默认状态不强调卡片底板。
- hover 时出现浅灰底板和更明显的质感反馈。
- 文案字号保持稳定，不用靠放大文字解决视觉问题。

## 8. 案例页通用规范

### 8.1 Hero 区

- 首屏不包卡片。
- 标题、导语、元信息和主图直接组织层级。
- 主图可以保留圆角，但不要再加厚重外壳。

### 8.2 正文区块

- 标题和导语优先通栏。
- 只有在内容本身需要图像承托时，才引入卡片或浅灰面板。
- 正文区块之间用留白与极细分隔线控制节奏，不靠重容器切段。

### 8.3 图片

- 图片要优先保证内容清晰，不因“留白感”被缩得过小。
- 当图文改成左右结构后，图片尺寸要能支撑右侧视觉重量，避免中间空太多。
- 如果原图背景过黑、过突兀，可以做背景统一处理，但不要额外改动画面信息。

## 9. Digistudio 当前落地规范

参考 [digistudio.html](/Users/yezehong/Documents/Codex/2026-05-31/files-mentioned-by-the-user-project/portfolio-site/digistudio.html)。

### 9.1 页面定位

`digistudio` 不是通用作品页模板，而是一页带“编辑器/画布”气质的案例页。它的重点不是秀视觉，而是把“复杂逻辑被重新组织清楚”这件事讲明白。

### 9.2 Hero

- Hero 采用左右非对称分栏，文字约 48%，主图约 52%。
- 标题拆成两层：
  - 主标题：`Digistudio UGC 对话系统编辑器`
  - 副标题：`把复杂分支对话改成一张能看懂、能继续改的画布`
- `Case Study` 使用等宽、深蓝、小尺度眉标。
- 顶部导航下方增加 1px 渐变线，作为页面视觉起点。
- 右侧主图放入 Mac 浏览器风格外壳，使用 1px 细线和低饱和阴影。

### 9.3 Process

- 使用 4 列横向流程结构。
- 编号使用浅蓝底的圆形徽标感。
- 节点间用轻量箭头/细线连接，强调叙事连续性。

### 9.4 Problem Definition

- 左侧为结构化问题说明，右侧为缩小后的旧版痛点图。
- 右图尺寸已经放大一轮，用于减少中部无效留白。
- 旧图原本纯黑背景过突兀，现已统一处理成更适合白色页面的冷灰蓝背景。
- 行业参考内容改成高亮分析框，不再作为普通正文平铺。

### 9.5 User Journey

- 标题和导语必须通栏，不能再拆成左右分栏。
- 4 张卡片下方用一条横向行为线串起来。
- 卡片内文案要能看出主语，明确是“创作者在做什么”。
- 用户原声不用斜体，改用浅灰字和中文引号。
- 底部痛点说明统一底线对齐，不能被单张卡片字数撑坏。
- 底部微标统一换成项目深蓝系精致标记，不再使用红色警示感符号。
- 卡片需要轻微呼吸感阴影，避免陷进白底。

### 9.6 Phase 01

- 标题和导语采用通栏。
- 结构为：上方说明，下方左右双栏。
- 左侧不是普通并列卡片，而是可切换的 Tab。
- 当前激活项左侧用深蓝竖线提示。
- 右侧 GIF 面板使用浅灰底板和轻画布纹理。

### 9.7 Phase 02

- 标题和导语同样必须通栏。
- 下方采用 3 列矩阵。
- 卡片默认是淡灰无边底板。
- hover 时收紧为浅蓝细边，并给非常轻的 halo 效果。
- 静态截图只展示局部关键细节，不再塞整张编辑器大图。

### 9.8 结果区

- 标题和导语采用通栏。
- 三个结果卡片等宽排布。
- 关键数字使用蓝色渐变字。
- 卡片之间只用极细垂直分隔线，不用厚重色块。

### 9.9 收束区

- 保持简洁，以结论性表达收尾。
- 不再额外加重卡片包装。

## 10. 文案与视觉协同规则

这轮调整已经证明，文案密度会直接影响页面视觉质量，因此后续要同时遵守下面两条：

- 标题先讲清楚解决了什么，不先讲抽象方法论。
- 正文段落优先短一点，让留白和节奏一起成立。

对于 `digistudio`，当前文案方向已经固定为：

- 偏面试讲述
- 保留专业判断
- 但不堆大厂黑话
- 第一人称只在关键判断和取舍处少量出现

## 11. 后续迭代时不要回退的点

- 不要把页面底色改回偏黄或偏米。
- 不要重新把大量正文塞进卡片里。
- 不要让章节标题和导语再次被左右拆栏，导致导语变窄换行。
- 不要把图片为了“显得轻”缩得太小，造成图文之间大面积空白。
- 不要重新引入过重描边、过厚阴影或层层套卡片。
- 不要让动效变成页面主角。
