# 项目封面 Brief

目标：每个项目封面都只表达一个最强记忆点，让访客在 2 秒内看懂“这是什么项目”。

## 01. DIGIMI UGC 流程编排工具设计

- 封面核心：可视化工作区里的流程关系
- 推荐来源页：[page-04.png](/Users/yezehong/Documents/Codex/2026-05-23/files-mentioned-by-the-user-md/assets/pdf-pages/page-04.png)
- 最适合截取的内容：右侧蓝色工作区大图，保留节点卡片、连接线、分支走向、底部创建入口
- 不建议保留：左侧大段案例说明、右上和右下信息卡
- 推荐构图：让黄色高亮节点落在偏左中位置，右侧保留两到三个分支节点，形成“从一个节点分流出去”的阅读路径
- 推荐背景气质：浅紫、浅蓝或奶白，强调“专业工具但不压迫”
- 这张封面想传达的句子：这是一个把复杂分支关系整理得可视、可编辑、可扩展的专业工具

## 02. 全球电影数据可视化设计

- 封面核心：地球主视图的沉浸感
- 推荐来源页：[page-20.png](/Users/yezehong/Documents/Codex/2026-05-23/files-mentioned-by-the-user-md/assets/pdf-pages/page-20.png)
- 最适合截取的内容：右侧地球大图，只保留球体和一点空间背景
- 不建议保留：左侧标题段落、左下总结卡、底部三栏总结
- 推荐构图：让地球放大到几乎撑出画面，球体切边，保留欧洲和非洲区域，形成“信息先成为空间”的感觉
- 推荐背景气质：暖白或沙色底，避免太冷，保持作品集整体柔和
- 这张封面想传达的句子：这是一个把全球电影数据做成可探索空间体验的项目

## 03. 动态界面可读性研究

- 封面核心：高动态场景中的可读性对比
- 推荐来源页：[page-21.png](/Users/yezehong/Documents/Codex/2026-05-23/files-mentioned-by-the-user-md/assets/pdf-pages/page-21.png)
- 最适合截取的内容：右侧大场景游戏画面，加一个左上或右上的小 inset 对比框
- 不建议保留：左侧说明卡、底部长文本、页面级标题
- 推荐构图：大图占 80% 以上，小图只做“固定色 vs 自适应”的证据提示，不要拼太多研究材料
- 推荐背景气质：浅米色或偏灰绿，和画面里的荧光绿场景形成对比
- 这张封面想传达的句子：这不是普通界面设计，而是一个把动态场景可读性变成可验证实验的问题研究

## 04. 比邻星球多端体验优化

- 封面核心：游戏世界感 + 系统体验感
- 推荐来源：优先找真实游戏截图，而不是继续用纯说明型插画
- 最适合展示的内容：一个最能代表“建造 / 庄园 / 社交 / 多端体验”的场景画面，最好同时带一点 HUD 或交互痕迹
- 不建议展示：只有标签、只有文案、纯概念图标、过于平的品牌插画
- 推荐构图：主场景占满画面，局部保留角色、建筑或家园布局；如果有 UI，就只留一层轻 UI，不要做成后台界面
- 推荐背景气质：柔和蓝绿、草木色、奶油白，更像“可玩的世界”而不是“系统汇报”
- 这张封面想传达的句子：这是一个围绕玩家建造、社交和经营体验做系统梳理与优化的项目

## 统一规则

- 每张封面只保留一个视觉重点，不做汇报页截图直出
- 优先用成果画面，不用大段说明文字
- 允许保留少量辅助元素：一个局部放大框、一个轻标签、一个角标
- 封面里的文字越少越好，标题和分类已经在卡片下方出现
- 如果原始页面信息太满，优先裁掉页面说明区，只留下最强视觉区
